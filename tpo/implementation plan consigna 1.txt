# Implementation Plan - Extend Ecosystem with Order Service (Consigna 1)

Extend the Spring Boot microservices ecosystem by adding a new microservice called `order-service`. The new service will register with Eureka, be secure (requiring JWT validation via `auth-service`), expose REST endpoints, publish an `order-created` event to RabbitMQ, and consume a `product-created` event from RabbitMQ (originating from `inventory-service`). Additionally, `notification-service` will consume the `order-created` event to notify about newly placed orders.

## User Review Required
- **No breaking changes** are expected. Existing endpoints and routing remain intact.
- **Port assignment**: `order-service` will run on port `8085`.
- **Profiles**: The implementation will configure both `rabbitmq` and `kafka` support, but default to `rabbitmq` as requested by Consigna 1.
